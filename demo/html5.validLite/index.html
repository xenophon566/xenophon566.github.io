Getting Started

Common function in validLite plugin

validLite.getValidata(this); //individual validate

validLite.getValidata(); //overall validate

validLite.clearValidata(this); //clear individual message and class

validLite.clearValidata(); //clear overall message and class

For example individual validate

<input type="text" name="require" onBlur="validLite.getValidata(this);"/>

For example overall validate

<input type="button" value="Submit" onClick="validLite.getValidata();"/>

Plugin parameters setting method

$(function(){
   $("#myform").validLite({
      pattern: {
        number: /^\d+$/		//validate pattern (Default: myPattern)
      },
      setting: {
        getValid: '#myform',	//setting the validate scope (Default: body)
        corMsgShow: true,	//show an correct message (Default: false)
        errMsgShow: true,	//show an error message (Default: true)
        corClassShow: true,	//show an correct class (Default: false)
        errClassShow: true,	//show an error class (Default: true)
        corClassName: 'correctClass',	//correct class name
        errClassName: 'errorClass',	//error class name
        corElem: 'span',	//create an element of correct message
        errElem: 'span',	//create an element of error message
        corElemClass: 'corElemClass',	//class of correct element
        errElemClass: 'errElemClass'	//class of error element
      }
   });
});

      },
      "mode": "false"               //Optional

A case study in error mesage

1. Pseudocode of show message 
if( setting.errMsgShow == true ) {
   if( data.errId == !empty )
      show message
} else Do not show error message

2. Pseudocode of show class 
if( setting.errClassShow == true ) {
   if( data.errClass == !empty ) {
      displays the custom style that defined in CSS stylesheet
   } else if( setting.errClassName ) {
      displays the common style that defined in CSS stylesheet
   } else {
      Do not show error class
   }
} else Do not show error class

3. Pseudocode of show message element 
if( element of data.errId exists ) {
   show the element
} else {
   Create a new element ( defined in setting.errElem )
   and setting the id to "data.errId", class to "setting.errElemClass"
   and insert message to element
}

4. Pseudocode of single validate 
validate result = validLite.getValidata(element)

My default patterns

// validate the required field
require: /.+/
// validate the email field
email: /^\w+([-+.]\w+)*@\w+([-.]\w+)*\.\w+([-.]\w+)*$/
// validate the cellphone field
mobile: /^09[0-9*]{8}$/
// validate the visa card number field
visa: /^[0-9*]{4}$/
// validate the ID card number field
idcard: /^[A-Z]{1}[12]{1}\d{8}$/
// validate the Taiwan residence permit field
permit: /^[A-Z]{1}[ABCD]{1}\d{8}$/
// validate the Taiwan address field field
address: /[0-9*]/
// validate the Taiwan telephone field field
telphone: /^[0-9]{2,3}-[0-9]{3,4}-[0-9]{4}$/
// validate the Taiwan company tax ID field
vatnumber: /^\d{8}$/
// validate the english field
english: /^[A-Za-z]+$/
// validate the chinese field
chinese: /^[\u0391-\uFFE5]+$/
// validate the integer field
integer: /^[-\+]?\d+$/
// validate the decimal field
decimal: /^[-\+]?\d+(\.\d+)?$/
// validate is that safe field
unsafe: /^(([A-Z]*|[a-z]*|\d*|[-_\~!@#\$%\^&\*\.\(\)\[\]\{\}<>\?\\\/\'\"]*)|.{0,5})$|\s/

NOTE

1. If( data.errId == alert )    then you will get a POP-up window to show your message.

2. "data-validstatus"    Define the DOM status, do not need any operating.

3. You can insert a custom element to display the message as follow : "Attention: Element set to invisible"

<div> Validation Field Title <input type="text" data-validata='{...}'/> <span id="err" class="errClass" style="display:none;">This is a message</span> </div>

4. "data-validata"    In the validate case, setting the pattern is required and the other parameters are optional.

5. You can modify the default pattern to a appropriate pattern directly.

6. Optional parameter "mode" is a anti-validate setting, when setting to "false" then opposite to the validation pattern is True.
